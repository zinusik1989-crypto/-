# НейроФото — мобильное приложение для нейрофотосессии

Интерактивное Android-приложение (Kotlin + Jetpack Compose + Material 3) для ниши «нейрофотосессия».
Пользователь выбирает стиль, «генерирует» серию портретов, собирает галерею и избранное,
а интерфейс и набор категорий полностью настраиваются.

## Возможности

- **Нижняя навигация** (Material 3 `NavigationBar`): Студия · Галерея · Избранное · Настройки.
- **Две темы**: светлая и тёмная + режим «Системная». Плюс динамический цвет (Material You, Android 12+).
- **Пользовательское соглашение**: обязательный экран принятия при первом запуске и просмотр из настроек.
- **Полная настройка категорий**: добавление, редактирование (название, эмодзи, описание, цвет),
  включение/выключение, изменение порядка, сброс к стандартным.
- **Настройка интерфейса**: акцентный цвет, число колонок в сетке, скругление карточек.
- **Загрузка фото**: выбор из галереи (системный Photo Picker, без лишних разрешений) и съёмка на камеру (через `FileProvider`).
- **Реальная нейрогенерация**: подключение к API [Replicate](https://replicate.com) (image-to-image по фото + промпт из категории) с опросом статуса и сохранением результата.
- **Демо-режим** (по умолчанию): офлайн-имитация генерации, работает без интернета и ключей.
- **Локальное хранение** всех данных через DataStore (категории, кадры, настройки, ключ API, факт принятия соглашения); сами изображения сохраняются в приватную память приложения.

## Технологии

- Kotlin 1.9.24, AGP 8.5.2, Gradle 8.7
- Jetpack Compose (BOM 2024.06.00), Material 3, Navigation Compose
- DataStore Preferences, kotlinx.serialization, Coroutines
- Coil (загрузка изображений), OkHttp (API нейросети)
- minSdk 26, targetSdk/compileSdk 34

## Подключение нейросети (Replicate)

По умолчанию работает **демо-режим**. Для реальной обработки фото:

1. Зарегистрируйтесь на [replicate.com](https://replicate.com) и создайте токен: **Account → API tokens**.
2. В приложении: **Настройки → Нейросеть (API)** → выберите **Replicate** и вставьте ключ.
3. При необходимости измените модель (формат `owner/name`). По умолчанию используется
   модель редактирования изображений по фото и текстовому описанию стиля.
4. На вкладке **Студия** выберите стиль, загрузите фото (галерея/камера) и нажмите **Сгенерировать**.

> Разные модели Replicate ожидают разные имена входных полей. Клиент отправляет `prompt` и
> `input_image` (как data-URI). Если выбранная модель использует другое поле для изображения,
> скорректируйте `ReplicateClient.generate(...)` в `data/network/ReplicateClient.kt`.

Запрос идёт на endpoint моделей `POST /v1/models/{owner}/{name}/predictions` с заголовком
`Prefer: wait`, далее (при необходимости) опрашивается статус до готовности, и итоговое
изображение скачивается в приватную память устройства.

## Как получить APK

### Вариант 1. Автоматически через GitHub Actions (рекомендуется, Android Studio не нужен)

1. Создайте репозиторий на GitHub и запушьте в него этот проект:
   ```bash
   git init
   git add .
   git commit -m "НейроФото: первичная версия"
   git branch -M main
   git remote add origin <URL вашего репозитория>
   git push -u origin main
   ```
2. Откройте вкладку **Actions** — workflow `Build APK` запустится автоматически.
3. После завершения скачайте артефакт **NeuroPhoto-debug-apk** — внутри `app-debug.apk`.
4. Перенесите APK на телефон и установите (разрешите установку из неизвестных источников).

### Вариант 2. Локально в Android Studio

1. Откройте папку проекта в Android Studio (Giraffe или новее).
2. Дождитесь синхронизации Gradle (IDE сама докачает `gradle-wrapper.jar` и SDK-компоненты).
3. `Build → Build Bundle(s) / APK(s) → Build APK(s)`.
4. APK появится в `app/build/outputs/apk/debug/`.

### Вариант 3. Локально из консоли

Нужны JDK 17 и Android SDK (переменная `ANDROID_HOME`). Если установлен Gradle:
```bash
gradle wrapper --gradle-version 8.7   # один раз, чтобы создать wrapper
./gradlew assembleDebug               # Windows: gradlew.bat assembleDebug
```
Готовый файл: `app/build/outputs/apk/debug/app-debug.apk`.

## Структура проекта

```
app/src/main/java/com/neuro/photostudio/
├─ MainActivity.kt
├─ data/
│  ├─ Models.kt, SampleData.kt, AppRepository.kt   модели, дефолты, DataStore
│  └─ network/ReplicateClient.kt                   клиент API нейросети
├─ util/ImageUtils.kt                              фото: копирование, base64, загрузка, камера
└─ ui/
   ├─ AppViewModel.kt                              генерация, настройки, состояние
   ├─ theme/        светлая/тёмная темы, акцент, динамический цвет
   ├─ components/   карточки (Coil), свотчи, пустые состояния
   ├─ navigation/   нижняя навигация + гейт соглашения
   └─ screens/      Студия, Генерация, Галерея, Избранное, Настройки, Категории, Соглашение
```

## Примечание

В демо-режиме «генерация» — офлайн-имитация: если вы выбрали фото, оно появится в галерее под
выбранным стилем, иначе рисуется стилизованный градиент. Реальная обработка включается
подключением Replicate (см. раздел выше).
